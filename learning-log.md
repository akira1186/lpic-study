###### lspciコマンドが見つからないエラー

\###状況

`lspci`コマンドを実行しようとしたが、インストールされていないため実行できなかった。

\###解決方法

`pciutils`パッケージをインストールすることで解決した。

sudo apt install pciutils

###### SysVinitとsystemdの違いを確認

\###状況

教科書にあった`/etc/inittab`を探したが、ファイルは見つからなかった。

\###結果

自分の環境は`systemd`という新しい仕組みを使っているため、古い`SysVinit'の設定ファイルは存在しないことが分かった。



###### systemdのターゲット確認

\###試したこと

`ls -l /etc/system/system/default.target`を実行

\###結果

`No such file or directory`エラーとなった

\###わかったこと

初期設定ファイルは`/lib/system/system/`にある場合が多く。`systemctl get-default`コマンドを使えば端的に確認可能だと分かった。

###### 出力先（標準出力）のルール

\###デフォルト

コマンドを打つだけで画面に出る

\###リダイレクト(>)

そのコマンドの設定の時だけ、出力をファイルに横取りさせる。

\###分岐(tee)

`| teeファイル名`を使うと、画面とファイルの両方に流せる。



###### $ man manと打った

\###状況

$ man manと入力し、何を表しているのか、どう元の画面に戻るのかわからなくなった

\###結果と分かったこと

`man\[コマンド名]`でそのコマンドの詳しい使い方が見られる。今現在の画面は辞書のようなもの。`q`で終了することができる。

そのほかにもspaceキーで次のページに進む、bキーで前のページに戻る、/+単語でマニュアル内を検索することができることが分かった。



###### ディレクトリの作成から削除までを行ってみた

\###1.ディレクトリの作成

`mkdir top/second/third`を実行したが失敗し、親フォルダがないとその中の子は作れないことが分かったため、`mkdir top/second/third`を実行した。

さらに権限を絞ったフォルダも作成し、`mkdir -m 700 mydir`を実行した。

\###2 ファイルを動かして名前の変更

`touch memo.txt`でファイルを作り、`mv memo.txt top/second/third`で移動させ、`mv top/second/third/memo.txt top/second/third/complete.txt`で名前を変更した。

\###3削除

`rm -r top`を実行し、`top/second/third`が消えた。`-r`はとても強力であることが分かった。
