###### lspciコマンドが見つからないエラー

\###状況

`lspci`コマンドを実行しようとしたが、インストールされていないため実行できなかった。

\###解決方法

`pciutils`パッケージをインストールすることで解決した。

sudo apt install pciutils



###### lsusbコマンドが見つからないエラー

\###状況

`lsusb`コマンドを実行しようとしたが、インストールされていないため実行できなかった。

\###解決方法

`usbutils`パッケージをインストールすることで解決したが、何も変化がなく、それは環境が仮想的なものでUSB機器が一つも繋がっていない状態だからであると分かった。



###### SysVinitとsystemdの違いを確認

\###状況

教科書にあった`/etc/inittab`を探したが、ファイルは見つからなかった。

\###結果

自分の環境は`systemd`という新しい仕組みを使っているため、古い`SysVinit'の設定ファイルは存在しないことが分かった。



###### systemdのターゲット確認

\###試したこと

`ls -l /etc/system/system/default.target`を実行

\###結果

`No such file or directory`エラーとなった

\###わかったこと

初期設定ファイルは`/lib/system/system/`にある場合が多く。`systemctl get-default`コマンドを使えば端的に確認可能だと分かった。



###### サービス一覧の確認

\###状況

sudo systemctl list-unit-files -t serviceを実行

\###学んだこと

とても長い表示と最後にENDと表示されたが、`q`を押して終了してプロンプトに戻れるということが分かった。

自動起動、手動起動、絶対に起動させないという強い設定があることや、現在175個ものサービスが管理されていることも分かった。
